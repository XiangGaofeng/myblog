入门

脚本以#!/bin/bash 开头，**#!/bin/bash是指此脚本使用/bin/bash来解释执行。#!是一个特殊的表示符，其后，跟着解释此脚本的shell路径**。当然也可以使用其他的解释器路径。如#!/bin/bash、#!/bin/sh。

只有文件的第一行可以加#!，其他行加的会被认为是注释。shell文件的注释分为单行和多行注释

变量

系统预定义变量

在Linux中，有许多预定义的变量可用于Shell脚本和命令行操作。以下是一些常见的预定义变量：

  1. $HOME: 当前用户的主目录路径。
  2. $USER: 当前用户名。
  3. $PWD: 当前工作目录的完整路径。
  4. $PATH: 一个包含所有可执行文件路径的冒号分隔列表。
  5. $SHELL: 当前用户使用的Shell名称。

变量定义规则

在Linux中，变量定义必须遵循以下规则：

  1. 变量名称必须以字母或下划线字符开头，不能以数字开头。
  2. 变量名称只能包含字母、数字和下划线字符。
  3. 变量名称区分大小写。
  4. 变量名称不能与Shell关键字相同，例如if、else、do等。
  5. 变量值可以是任何字符串，包括空字符串。
  6. 如果变量值包含空格或其他特殊字符，则需要将其放在引号中。

特殊变量

在Linux中，有一些特殊变量始终可用于Shell脚本和命令行操作。以下是一些常见的特殊变量：

  1. $0: 当前Shell脚本的名称。
  2. $1、$2、$3 等：表示传递给Shell脚本或函数的参数，其中 $1 表示第一个参数，$2 表示第二个参数，以此类推。
  3. $#: 传递给Shell脚本或函数的参数数量。
  4. $@: 传递给Shell脚本或函数的所有参数列表。
  5. $?: 上一个命令的退出状态码。如果这个变量的值为 0，证明上一个命令正确执行；如果这个变量的值为非 0（具体是哪个数，由命令自己来决定），则证明上一个命令执行不正确了。
  6. $$: 当前Shell进程的进程ID。
  7. $!: 后台作业的进程ID。

这些特殊变量可以帮助您编写更灵活和通用的Shell脚本和命令行操作。例如，通过使用 $1 和 $2 变量，您可以将参数传递给Shell脚本，并根据这些参数执行不同的操作。通过使用 $? 变量，您可以检查最后一个命令的退出状态码，并根据返回值采取适当的措施。

除了这些常见的特殊变量外，还有许多其他的特殊变量可用于定制Shell的行为和设置。

流程控制

在Bash中，您可以使用控制结构来控制程序的执行流程。以下是一些常见的控制结构：

  1. if-then 结构：如果指定条件为真，则执行某些操作。
  2. if-then-else 结构：如果指定条件为真，则执行一个操作；否则，执行另一个操作。
  3. for 循环结构：重复执行一个操作，直到满足特定条件。
  4. while 循环结构：重复执行一个操作，直到不再满足特定条件。

正则表达式

正则表达式是一种文本模式，可以用来描述匹配某种格式的字符串。它由普通字符（例如字母、数字和符号）以及特殊字符（元字符）组成。在Linux中，常用的正则表达式工具包括grep、sed和awk。

常规匹配

正则表达式中的常规匹配是指模式中使用的普通字符。它们会按照文字字面意义进行匹配，除非它们被特殊字符所替代。以下是一些常规匹配示例：

  • 字母和数字：a-z、A-Z、0-9
  • 匹配单个字符：.（点号）
  • 匹配空白字符：\s、\t、\n
  • 匹配字符集合：[abc]、[^abc]
