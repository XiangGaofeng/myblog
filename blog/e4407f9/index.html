springboot读取配置文件的值并对bean做出修改

发表于2023-06-26|更新于2023-06-29|Java

springboot读取配置文件的值并对bean做出修改

为什么通过修改application.properties就能对IOC容器中的组件做出同步修改。比如为什么application.properties中server.port=8080，整个springboot项目就在8080端口启动。

首先需要说明一下@EnableConfigurationProperties和@ConfigurationProperties两个注解

@ConfigurationProperties

将配置文件中的属性值绑定到一个 Java 对象上,一般配合@Component注解使用

@EnableConfigurationProperties

@EnableConfigurationProperties注解来启用@ConfigurationProperties注解的配置类，并将其注册到Spring容器中。假设在一个类上有这样的标注，@EnableConfigurationProperties需要与@Configuration配合

即使你的ServerProperties类不在容器中，他也会给你自动注入到IOC容器中。

所以@EnableConfigurationProperties和@ConfigurationProperties是一组获取配置文件信息的注解。

看个例子

Person类，标注了@ConfigurationProperties(prefix = “person”)，相当于与 application.properties前缀为person的绑定

Test类@EnableConfigurationProperties(Person.class)和@Configuration

示例

以application.properties中的server.servlet.encoding.charset=UTF-8为例子

  1. 在配置文件中添加server.servlet.encoding.charset=UTF-8

  1. 在HttpEncodingAutoConfiguration类中，可以看到一个注解**@EnableConfigurationProperties(ServerProperties.class)**

  1. 进入ServerProperties类中，我们发现它的类上标记了**@ConfigurationProperties(prefix = “server”, ignoreUnknownFields = true)**

这样ServerProperties也能被注入到IOC容器中

ServerProperties中有这么一行代码

点击进入Servlet();，发现Servlet这个类还是在ServerProperties类中的。

点击Encoding（），进入Encoding类中

现在我们整理一下逻辑：

application.properties中有一行配置：server.servlet.encoding.charset=UTF-8

在HttpEncodingAutoConfiguration中有一行注解@EnableConfigurationProperties(ServerProperties.class)

ServerProperties类上标记了@ConfigurationProperties(prefix = “server”, ignoreUnknownFields = true)

ServerProperties有这么一行代码private final Servlet servlet = new Servlet();

Servlet类中有一行代码private final Encoding encoding = new Encoding();

Encoding类中有一个变量charset

所以server.servlet.encoding.charset=UTF-8对应的就是这个charset的值

进行一次debug，我们将server.servlet.encoding.charset设置为UTF-16

可以看到已经成功读取到了UTF-16。
