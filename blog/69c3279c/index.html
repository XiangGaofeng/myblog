撤销修改

假设你在工作区的文件中写了一些你不想要的东西。比如你喜欢学Java，但是你不想要了。先使用git status看看情况

注意这里有两行

这其实是Git在告诉你，如果你想提交的话，就使用git add将文件添加到暂存区，如果你想取消更改，那么使用git restore,来试试

很舒服，修改的东西没有了。当然，如果你直接使用文本编辑器删掉这一行文字也是没毛病的！

你也可以使用git checkout -- 文件名完成同样的功能。

git restore 文件名 和git checkout -- 文件名就是让这个文件回到最近一次git commit或git add时的状态。

一种是文件自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；

一种是文件已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。

现在，你又发现了，你将你不想告诉别人的话添加到了暂存区，那我怎么取消呢。

当我使用git add之后，使用git status之后

ok 可以看到已经变了

然后再使用git restore .\test.txt，直接舒服了！

如果你提交到了版本库，我们可以使用git reset --hard commit_id进行回退，但是如果你上传到了远程仓库，那就G了。

撤销修改汇总：

  • 情况1：文件只在工作区操作，未add。撤销操作：git restore 。结果：工作区文件回退。
  • 情况2：文件已add，未commit。撤销操作：git restore –staged 。结果：暂存区文件回退，工作区文件未回退，如需继续回退，操按情况1操作。
  • 情况3：文件已add，已commit。撤销操作：git reset –hard commit_id。结果：工作区文件、暂存区文件、本地仓库都回退

来个例子，当我们删除文件时，如果使用shift+删除，就是直接永久删除文件了，这时候可以通过git restore 被删除的文件名来对其进行恢复

分支管理

创建分支

创建分支使用git checkout -b 分支名

git checkout命令加上-b参数表示创建并切换，相当于以下两条命令：

用git branch命令查看当前分支,git branch命令会列出所有分支，当前分支前面会标一个*号。

删除分支

git branch -d 分支名

切换分支

git checkout 分支名

合并分支

git merge 分支名可以将分支合并到当前分支。

注意到上面的Fast-forward信息，Git告诉我们，这次合并是“快进模式”，也就是直接把master指向dev的当前提交，所以合并速度非常快。

当然，也不是每次合并都能Fast-forward，我们后面会讲其他方式的合并。

总结：

查看分支：git branch

创建分支：git branch <name>

切换分支：git checkout <name>或者git switch <name>

创建+切换分支：git checkout -b <name>或者git switch -c <name>

合并某分支到当前分支：git merge <name>

删除分支：git branch -d <name>
